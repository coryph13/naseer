# Сессия 02: Редизайн каталога (общий + по линейке)

## Цель
Переписать обе каталожные страницы (`/catalog` и `/catalog/[line]`) под editorial-дизайн главной. Каталог — ядро сайта, должен выглядеть «продумано», а не как сетка из конструктора.

## Контекст
- Главная и шапка уже переработаны. Использовать ту же дизайн-систему: `eyebrow`, `display-section`, `display-mono`, `body-lead`, `bg-cream`, `text-ink`, brand `#8f1538`.
- Данные: `data/lines.json` (5 линеек) + `data/products.json` (60 SKU, у всех есть `image`).
- Цвета линеек захардкожены в страницах сейчас — оставить как есть, не выносить в JSON ради одной задачи.
- Компонент `ProductsGrid.tsx` (`src/components/ProductsGrid.tsx`) используется на `/catalog/[line]` — его тоже надо переделать.

## Что делать

### 1. `/catalog` — `src/app/[locale]/catalog/page.tsx`

- Hero без brand-плашки. Вместо неё:
  - `eyebrow` сверху («Каталог»)
  - Крупный display-заголовок (например «Шестьдесят характеров.» / «Sixty characters.» / «Oltmishta xarakter.») — генерируй сам, копирайтер заменит.
  - Подзаголовок body-lead с описанием: «Пять линеек, шестьдесят SKU, один производитель.»
- Sticky-нав линеек: переоформить под чистый editorial-стиль. Кнопки без `bg-gray-100` плашек, просто текст с цветной точкой и счётчиком SKU. Активная — подчёркнутая полоска цвета линейки (через `layoutId`). Высота полосы — 1px.
- Каждая линейка-секция:
  - Header линейки: `eyebrow` с номером (01 / 05), потом большой display-section заголовок с названием линейки, потом короткое описание из `t('home.lines.<slug>')` или из `lines.json`.
  - **Большой hero-кадр линейки** — full-width image (около 60vh высотой) с overlay цвета линейки и кнопкой «Открыть линейку» поверх. Это тот же подход что в LineShowcase главной, но статичный.
  - Сетка SKU под ним: `grid-cols-2 sm:grid-cols-3 lg:grid-cols-4 xl:grid-cols-5` или `grid-cols-2 md:grid-cols-3 lg:grid-cols-4` — экспериментировать. Карточки SKU чище: белый фон, минимальная тень, без ярких градиентов. Цветная точка вкуса слева сверху (8px). Название и фасовка снизу. Hover — лёгкий lift + smooth border-brand на 1px.
  - Между линейками — большие пустые отступы (`py-24` или `py-32`).
- В Hero страницы — кнопка «Скачать прайс PDF» как secondary (опционально, можно скипнуть и оставить заметку TODO).

### 2. `/catalog/[line]` — `src/app/[locale]/catalog/[line]/page.tsx`

- **Full-bleed hero** с фотографией линейки (используя `lineImages[slug]`) на всю ширину, высота `~70vh` на desktop, `50vh` на mobile.
- Поверх фото:
  - `eyebrow` светлого цвета слева сверху: «Линейка 02 / 05»
  - Огромный display-mono заголовок с названием линейки (`text-7xl md:text-9xl`) внизу или по центру.
  - Тонкая разделительная полоска цвета линейки.
  - Описание линейки `body-lead` светлого цвета.
  - Breadcrumb «Каталог / Lollis» сверху, маленький, светлый.
- Под hero — **переключатель линеек** (как в LineShowcase главной, но без авто-смены): миниатюры стопкой или горизонтально. Активная — обведена цветом линейки.
- Фильтры — компактные, в одну строку. Eyebrow «Фильтры», потом chip-кнопки. Переоформить без layout-pill (у сейчас слишком игривое поведение). Можно оставить layoutId, но смягчить пружину.
- Сетка SKU — как у `/catalog`, но с большими карточками (`grid-cols-2 md:grid-cols-3 lg:grid-cols-4`). У каждой SKU — своя картинка `product.image` (есть у всех). Без `mixBlendMode: multiply` — это от старого. Просто `object-contain` или `object-cover` (выбрать что лучше смотрится с предметкой на нейтральном фоне).
- Модалка SKU (bottom sheet сейчас) — оставить, но почистить: убрать `flavor` маленький eyebrow, заменить на спокойный список характеристик в две колонки. Кнопка CTA внизу пусть будет brand-цвета.

### 3. `src/components/ProductsGrid.tsx`

Переписать в духе editorial:
- Без `motion-pill` для weight (или сделать спокойнее)
- Без gradient-blur оверлеев на карточках
- Чистая фотокарточка + минимум текста + цветная точка вкуса
- Modal: чище типографика, больше воздуха

### 4. i18n

Добавить ключи:
- `catalog.heroTitle`, `catalog.heroBody` — для нового hero общего каталога
- `catalog.lineN` где N это номер (для eyebrow «01 / 05» — на самом деле просто число, не локализуется)
- `catalog.openLine` — «Открыть линейку» / «Open line» / «Liniyani ochish»
- `catalog.filtersEyebrow` — «Фильтры» / «Filters» / «Filtrlar»
- `catalog.weight`, `catalog.flavor`, `catalog.all` — уже частично есть, проверь и добавь чего нет
- `catalog.detailsBtn` — «Подробнее» / «Details» / «Batafsil»
- `catalog.modal.weight`, `catalog.modal.packaging` — уже частично есть
- Тексты можно генерировать.
