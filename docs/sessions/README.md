# Sessions — план работ по сессиям

Сайт-каталог Naseer строится постепенно, каждая большая задача — отдельная сессия Claude Code (так не упирается в лимит токенов). В этой папке лежат self-contained планы каждой сессии.

## Как пользоваться

**В начале новой сессии** скажи Claude:

> Продолжи

Claude:
1. Прочитает `MEMORY.md` (auto-memory) и найдёт ссылку на `session_roadmap`.
2. Откроет `session_roadmap.md` и найдёт **первую сессию со статусом `pending`**.
3. Прочитает соответствующий файл `docs/sessions/0X-foo.md` — там цель, контекст, точные изменения, definition of done.
4. Выполнит работу.
5. Пометит сессию как `done` в roadmap.

**Если хочешь конкретную** — скажи: `Продолжи. Сессия 3` или `Сделай редизайн каталога`.

## Принципы (общие для всех сессий)

- **Никаких комментариев в коде** (см. memory `feedback_no_comments`).
- Соблюдать дизайн-систему из главной: классы `eyebrow`, `display-hero/section/mono`, `body-lead`, brand-цвета (`#8f1538`), `var(--color-cream)`, `var(--color-ink)`. Шрифты: Source Serif 4 (display) + Inter (body) — не менять без явной просьбы.
- `prefers-reduced-motion` уважать через `useReducedMotion()` framer-motion.
- i18n: любой новый текст — в `messages/{ru,uz,en}.json`. Тексты можно генерировать (копирайтер пройдётся позже).
- Цены НЕ показывать никогда.
- LSP-диагностики «Cannot find module» — известная проблема `node_modules` под root, не править.
- В конце сессии: пометить `done` в `session_roadmap.md`, кратко (1-2 предложения) сообщить пользователю что сделано.
