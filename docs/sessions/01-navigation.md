# Сессия 01: Шапка с телефоном + Footer-полировка

## Цель
Переделать `Header` под новую дизайн-систему главной (editorial, чистый), добавить телефон как заметный CTA. Привести `Footer` в порядок к новому стилю.

## Контекст
- Главная страница уже переработана (см. `src/app/[locale]/page.tsx`) — Header сейчас остался прежним, тёмно-красный, контрастирует с editorial-страницей и выглядит «из старого проекта».
- Шапка используется на ВСЕХ страницах, поэтому редизайн повлияет на всё. После этой сессии редизайн остальных страниц пойдёт чище.
- Текущий `Header.tsx` — `'use client'`, использует framer-motion `layoutId` для подсветки активной ссылки и переключателя локалей. Это оставить.
- Логотип: `/logo.svg`. Для светлой шапки нужна тёмная версия — если SVG подкрашиваемый через `currentColor`, отлично; иначе можно оставить как есть и проверить, как выглядит на белом.

## Что делать

### 1. `src/components/Header.tsx` — переписать
- **Фон:** белый/cream (`bg-white`), без brand-фона. Тонкая нижняя граница (`border-b border-ink/10`).
- **При скролле > 20px:** добавить `backdrop-blur` + лёгкую тень (`shadow-[0_2px_24px_-12px_rgba(20,17,15,0.18)]`). Без смены цвета фона.
- **Лого:** слева, чуть крупнее (h-10). Если `logo.svg` сейчас белый — нужна тёмная версия. Если файл белый/неуниверсальный, временно использовать text-логотип `<span className="display-section text-2xl">Naseer</span>` и добавить TODO в memory `[[project_catalog]]` про тёмный логотип.
- **Навигация (desktop):** по центру или ближе к правому краю. Серые ссылки (`text-ink/65`), активная — `text-ink` с тонкой подчёркивающей полоской снизу через `layoutId="nav-underline"` (вместо плашки-фона). Высота полоски 1px, цвет `bg-brand`.
- **Переключатель локалей:** компактный pill справа, на белом фоне. Активная локаль — `bg-ink text-white` (или `bg-brand text-white`), неактивные — `text-ink/55`. Сохранить `layoutId="locale-active"` для плавного перехода.
- **Телефон-CTA:** заметный pill справа от локалей. Формат: иконка телефона (тонкая SVG, не круглый чип) + номер `+998 99 115 03 02`. Класс примерно: `inline-flex items-center gap-2 bg-ink text-white pl-4 pr-5 py-2 rounded-full text-sm font-medium hover:bg-brand transition-colors`. На hover — лёгкое расширение `gap-2` → `gap-3` (как кнопки на главной). Это `<a href="tel:+998991150302">`.
- **Высота шапки:** h-18 (72px) для desktop, h-16 для mobile. Чуть больше чем сейчас, чтобы дать дыхание новому стилю.
- **Mobile:** телефон не помещается с номером — показать **только иконку** телефона как круглую кнопку. Меню-бургер рядом. Локали — внутри развёрнутого меню, не в основной строке (сейчас они мешают, особенно с длинной локалью «Ру»).
- **Развёрнутое мобильное меню:** белый фон, нав-ссылки крупные (`display-section text-3xl`), под ними — переключатель локалей и кнопка телефона. Анимация открытия — текущая (height + opacity).

### 2. `src/components/Footer.tsx` — освежить
- Фон оставить тёмный (`bg-ink` вместо `bg-brand` — будет современнее), либо `bg-cream` со светлым стилем. **Решить по контексту дизайна — рекомендация: `bg-ink text-white/70`**, brand-цвет уйдёт в акценты.
- Структура колонок: лого + короткая фраза (одна строка из `t('home.promise.body')` или похожее) | линейки списком | контакты | мини-блок «Работаем по будням, заявки до 17:00». Четыре колонки на desktop, по одной на мобиле.
- Editorial-типографика: `eyebrow` для заголовков колонок, `text-sm text-white/55` для пунктов.
- Низ футера: copyright + ссылка-mailto + ссылка-tel в одну строку, разделители — точки.
- Убрать ссылки `?line=...` (нерабочие сейчас в `/catalog`), заменить на `/${locale}/catalog/${slug}` — реальные страницы линейки.

### 3. Адаптация существующих страниц
- На большинстве страниц Header был фиксирован поверх brand-блока — после смены на белый шапка может теряться над собственным белым hero. Если так — у hero-секции на каждой странице есть `pt-22/24/28/32`. Этого достаточно. Не нужно ничего больше менять, кроме случаев когда Header сейчас визуально «садится» на тёмный hero — тогда тёмная страница (например, контакты hero `bg-[#8f1538]`) должна получать тёмную версию шапки. **Решение:** не делать тёмную версию ради одной секции, всё равно эти страницы переделываются дальше. Просто проверь, что белая шапка читаема над hero. Если плохо — добавь под Header тонкий белый отступ на затронутых страницах временно.

### 4. i18n
- Добавить ключ `nav.phone: "+998 99 115 03 02"` в `messages/{ru,uz,en}.json` (формат для разных локалей одинаковый, но через перевод можно будет поменять).
- Добавить ключ `nav.callUs` (для `aria-label`/screen-reader): `ru: "Позвонить"`, `en: "Call us"`, `uz: "Qo'ng'iroq qilish"`.
